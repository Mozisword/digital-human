# 2026-07-07 数字人系统功能扩展

## 本次新增
- **口型与讲话一致**：重构 Web Audio 图（masterGain → analyser → 扬声器；masterGain → MediaStreamDestination）。嘴型严格跟随播放音频的实时 RMS 振幅（TTS 与唱歌都用同一路径，自动同步）。
- **可下载视频**：🎬 录制视频按钮用 `canvas.captureStream(30)` + `MediaStreamDestination` 经 `MediaRecorder` 合成 webm 下载（完全客户端、离线）。
- **完全离线本地 TTS**：新增 Piper 引擎（设置里切 edge-tts / piper）。`/api/tts/piper/install` 一键下载中文音色（zh_CN-huayan-medium 等）。前端设置面板含音色下拉 + 下载按钮。
- **歌库 + 唱歌**：`/api/songs`（列表/上传/在线添加/音频流/歌词/删除）。前端🎵歌库弹窗可上传本地歌曲(音频+可选LRC)、添加在线直链URL、点“让数字人唱”播放并驱动口型，LRC 歌词同步显示。亦支持聊天输入 `/sing 歌名`。

## 关键技术点 / 坑
- Piper 正确 API：`voice.synthesize_wav(text, wav_file)`，其中 wav_file 是 `wave.open(buf,"wb")`（`synthesize` 只返回迭代器，不是写文件）。
- 中文 Piper 需 `g2pw`（已加入 requirements）。sandbox 内 `pip install g2pw` 已后台执行。
- **sandbox 屏蔽 huggingface.co**（连接超时），所以离线模型下载与 g2pw 模型数据在本环境无法拉取；用户本机有网可正常下载。
- sandbox 的 safe-delete 拦截 `Path.unlink()`（回收站不可用会抛 OSError）→ 已用 try/except 包裹删除逻辑，仅阻碍物理删文件、不影响元数据清理。
- 上传歌曲的 multipart 测试中 title 字段因手工拼接格式问题未生效，但前端用 FormData 正常。
- 重启服务用 `netstat -ano | grep :8000 | grep LISTEN` 取 PID + `taskkill /F /PID`；Git Bash 下 `//F` 会被改写，用单斜杠 `/F`。
